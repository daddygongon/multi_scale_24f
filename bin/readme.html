1 ~/bin

userが自分で使うcommandの置き場が~/binです． ここにあるいくつかのcommandをそこに置いて使ってください．

2 パスを通す

2.1 tldr(too long don't read)

fishで使えるようにパス(path)を通す必要があります．

  • emacs ~/.config/fish/config.fish
set PATH . ~/bin $PATH

とします．

ついでに，"."(dot, current directory, 今のdirectory)も追加しています．

2.2 $PATH

実行可能コマンドの置き場所を パソコンに教えてあげる必要があります． そうでないと，パソコンは全ディレクトリを探す 必要がありますよね？！ 現在のshell環境でどこにパス(path)が通っているかは，

  • echo $PATH

で確認できます．

ちなみに$で始まる大文字変数は 環境変数(environment variable) と呼ばれます．

config.fishを変更したら，それをfishに知らせる必要があります．

  • shellを新たに起動するか，
  • source ~/config/fish/config.fish

でできるはず．echoして確かめてください．

2.3 順番

$PATHはリストになっていて，前から順にsystemは実行可能ファイルを探します． whereis, echo $PATH で確認できます． 

bob@surfacepro9 ~/m/bin (main)> whereis open
open: /usr/bin/open /home/bob/bin/open /usr/share/...
bob@surfacepro9 ~/m/bin (main)> echo $PATH
. /home/bob/bin /usr/sbin /usr/bin /sbin /bin ....

昔はよくこれで路頭に迷ったな．．．

2.4 その他の置き場

systemが使うのや，そのほかのがあります．

  • /bin, /sbin,
  • /usr/bin
  • /usr/local/bin

apt(brew), gemなどでバリエーションがありますので， package managerのマニュアルを参照ください．
